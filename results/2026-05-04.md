### 4.2 Opus 4.7-specific contract divergence on Invoke API

Five tests where the three models do not all agree:

| Test | Opus 4.7 | Opus 4.6 | Sonnet 4.6 | Note |
| --- | :---: | :---: | :---: | --- |
| `structured_outputs`             | ⛔ | 🟢 | 🟢 | `output_config.format` rejected with "Extra inputs are not permitted" — Anthropic docs direct Opus 4.7 users to the Mantle endpoint |
| `strict_tool_use`                | ⛔ | 🟢 | 🟢 | Same Mantle-only deployment pattern as structured_outputs |
| `thinking_enabled_with_effort`   | ⛔ | 🟢 | 🟢 | Opus 4.7 introduces a new `thinking.effort` parameter surface unknown to Invoke API |
| `builtin_memory_tool`            | 🟢 | 🟢 | ⛔ | Reverse pattern: Sonnet 4.6 rejects (different gating decision) |
| `vision_multi_image`             | 🟢 | ❌ | 🟢 | Opus 4.6 transient failure (see §4.3) |

The **three Opus-4.7-only ⛔ cells** (structured_outputs, strict_tool_use,
thinking_enabled_with_effort) all share one underlying cause: *Anthropic
deployed those features on the Mantle endpoint
(`bedrock-mantle.{region}.api.aws/v1/messages`) but did not back-port them
to the Invoke API path on `bedrock-runtime`*. This suite intentionally
measures only the Invoke API and faithfully reports the rejections that
default Bedrock users (Claude Code, AnthropicBedrock SDK, boto3) actually
encounter.

### 4.3 ❌ Open failure: `vision_multi_image` on Opus 4.6

The single ❌ in this run. Test source:

- Test-side contract assertion is too strict for Opus 4.6's response
  variance.

Action item: re-examine `test_image_multi.py` — ensure the assertion
allows Opus 4.6's natural response phrasing while still detecting the
two-image grounding. Tracked as future work.

### 4.4 Mantle scope reminder (carried from this session's discussion)

This run measures only `bedrock-runtime` Invoke API. The Mantle endpoint
is **opt-in** at every layer:

- **Claude Code**: requires `CLAUDE_CODE_USE_MANTLE=1` (separate env var,
  mutually exclusive with `CLAUDE_CODE_USE_BEDROCK=1`). Default Claude
  Code on Bedrock = Invoke API.
- **SDK / boto3 / custom apps**: requires explicit base-URL change to
  `bedrock-mantle.{region}.api.aws` plus the Anthropic Messages-API
  request shape (`/v1/messages?beta=true`).
- **Region availability**: Mantle is deployed in 13 regions only;
  `ap-northeast-2` (this suite's region) is **not** one of them.
- **Account allowlist**: model-by-model gating applies even where Mantle
  is deployed.

So the ⛔ cells in §4.2 are not theoretical — they reflect the contract
that ~100% of `ap-northeast-2` Bedrock users on Opus 4.7 will hit unless
they make a deliberate region+endpoint switch. See
[`docs_vs_reality.md`](docs_vs_reality.md) §"Configuration notes" and
[`prompt_caching_verified.md`](prompt_caching_verified.md) §"Quick
reference — Claude Code env vars" for the consolidated reference.

| --- | ---: | ---: |
| Fresh input (non-cached) | 698,143 | 56.3% |
| Cache create — 5m       | 213,507 | 17.2% |
| Cache create — 1h       |  57,109 |  4.6% |
| Cache create — total    | 270,616 | 21.8% |
| Cache read              | 270,616 | 21.8% |
| **Total billable input** | **1,239,375** | 100% |

**Cache write : read = 1 : 1**, by design. Most cache tests are
cold-start (one fresh write) plus a single hot read to verify the prefix
came back. Tests that don't exercise caching contribute only to the fresh
input bucket. The 1h share (~4.6% of billable input) corresponds to the
two cache TTL tests (`cache_ttl_1h` and `cache_ttl_mixed_5m_and_1h`)
plus their cross-model expansion.

## 7. Conclusions

1. **Contract baseline stable.** The 12 / 9 / 10 ⛔ counts are unchanged from
   the prior baseline — no Anthropic-side or Bedrock-side regressions on
   the Invoke API surface as of 2026-05-04 in `ap-northeast-2`.

2. **1h prompt cache works as documented in this project's findings.**
   Re-verified end-to-end across all 3 models. The `ttl: "1h"` field on
   `cache_control` populates `ephemeral_1h_input_tokens`; the docs-named
   beta header remains rejected.

3. **Opus 4.7's three Invoke-only ⛔ cells reflect a real deployment gap.**
   `structured_outputs`, `strict_tool_use`, and `thinking_enabled_with_effort`
   are Mantle-only on Opus 4.7 — confirmed expected behavior, not a bug
   in our suite. Default Bedrock users will see these rejections.

4. **One ❌ to investigate.** `vision_multi_image` on Opus 4.6 fails
   reproducibly — likely test-contract issue rather than API issue. To be
   triaged separately.

5. **Cost stayed within the README's `~$5 USD per matrix run` ballpark**
   for the lower-priced two models; Opus 4.7 alone consumes the lion's
   share due to 1M-context test inclusion. Exact dollar amount depends on
   current Bedrock pricing for `ap-northeast-2`; token totals are above
   for direct calculation.
